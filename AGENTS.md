# react-native-nitro-ark

- This project is a pure C++ nitro module for providing FFI bindings to react-native for Ark (a Bitcoin layer 2 protocol).

Important directories:

- `bark-cpp`: The Rust FFI code which calls Ark APIs and exposes them via FFI to C++ side using cxx. We make wrapper for most methods inside `lib.rs` and FFI definitions inside `cxx.rs`.
- `logger`: A simple lib for initializing Android and iOS logging.
- `react-native-nitro-ark/cpp/generated`: Header files automatically generated by cxx.
- `bark-cpp/build-android.sh`: Builds Android binaries and copies them in the right location including the header files.
- `bark-cpp/build-ios.sh`: Builds iOS binaries and copies them in the right location including the header files.
- `react-native-nitro-ark/cpp/NitroArk.hpp`: The C++ code calling the Rust code via FFI.
- `react-native-nitro-ark/src/NitroArk.nitro.ts`: Typescript definitions which we define for nitro-module to output C++ glue code.
- `react-native-nitro-ark/src/index.tsx`: Methods to expose to the JS side of react-native.
- `react-native-nitro-ark/nitrogen/generated`: The automatically generated code from typescript definitions by running `yarn nitrogen` inside `react-native-nitro-ark` directory.
- `react-native-nitro-ark/example/src/App.tsx`: Example app for this project.

Important notes:

- The Ark APIs generally return complex types and for cxx to handle them we have to manually do type conversion into simpler types.
- Always try to match the struct type with a corresponding simple type by converting to a string or a number, etc. Sometimes it is not possible, in such a case just the ask the user what to do.
